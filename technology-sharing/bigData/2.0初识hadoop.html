# 硬盘出错

  • 没有出错的硬盘，提升硬盘的稳定性成本较高
  • 提供健康监控、快速检测、自动恢复到HDFS

# 流式数据访问

  • 适应批处理处理的要求
  • 交互式处理、随机访问支持差

# 简单的一致性

  • 一次写入、多次读取的访问模式
  • 一个文件一经创建、写入、关闭后，不需要改变
  • 简单数据一致性的实现，实现了高吞吐的数据访问

# 大规模数据集

  • 适合存储GB TB的大文件

# 移动的计算

  • 计算移向数据，计算本地化

# HDFS 分布式文件系统

  • 本质上是分散式，大文件切割存储。（切蛋糕）

# MapReduce

  • 分布式离线批处理计算框架

# NomeNode

  • 单个实例的主节点
  • 为Client提供读写服务，本质上客服端是和DataNode 完成读写操作的
  • 元数据管理

# fsimage 文件

  • hdfs-site.xml/dfs.name,dir
  • 印像文件是内存metadata数据快照
  • 不会保存Block的位置信息

# edits log

  • 记录增、删、改名、移动操作

# SecondNameNode

  • 辅助结点，不是备份结点！

  • 连接到NameNode,完成fsimage文件和edits log文件合并，因为需要耗时长，消耗资源，NameNode并不合适，所以分出去让其他进程来做。

  • 合并策略

    1、时间 2、大小

  • check point 一个时间点

# DataNode

  • 数据结点
  • block存储的节点
  • 通过心跳定时向NameNode汇报监控状态
  • 定时汇报自己拥有的块信息
