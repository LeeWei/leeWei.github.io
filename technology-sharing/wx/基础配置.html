# 全局json配置

app.json 是小程序的全局配置,包含小程序所有的页面路径、界面表现、网络超时时间、底部 tab 等

{
  "pages": [
    "pages/index/index",
    "pages/logs/index"
  ],
  "window": {
    "navigationBarTitleText": "Demo"
  },
  "tabBar": {
    "list": [{
      "pagePath": "pages/index/index",
      "text": "首页"
    }, {
      "pagePath": "pages/logs/index",
      "text": "日志"
    }]
  },
  "networkTimeout": {
    "request": 10000,
    "downloadFile": 10000
  },
  "debug": true,
  "navigateToMiniProgramAppIdList": [
    "wxe5f52902cf4de896"
  ]
}

# 页面配置

在app.json 中可以配置小程序全部的设置，同样的在每一个页面下也可以单独配置.每一个页面都存在对应的文件夹，对应的下有 .json 的文件，该文件就是对当前页面的配置。当app.json 和 文件下.json 下的配置冲突时，微信小程序会默认以该文件下的.json 配置文件为准。

{
  "navigationBarBackgroundColor": "#ffffff",
  "navigationBarTextStyle": "black",
  "navigationBarTitleText": "微信接口功能演示",
  "backgroundColor": "#eeeeee",
  "backgroundTextStyle": "light",
  "usingComponents": {}
}

# 微信小程序的生命周期

  • app.js --->同app.json一样整个微信小程序只有一个app实例，开发过vue的同学应该很清楚，这和vue 是相似的。
App({
  onLaunch (options) {
    // 监听页面加载
  },
  onShow (options) {
    // 监听页面显示
  },
  onHide () {
    // 监听页面隐藏
  },
  onError (msg) {
    // 错误监听函数。
  },
  globalData: 'I am global data'// 全局存储，类似vue的store
})
