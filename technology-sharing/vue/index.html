# 为什么要学vue

Vuejs作为国人开发的一款前端框架，不仅有强大的社区支持，很多API 都是中文的，学习难度相对小。而且现在越来越多的大厂都开始使用vue,本模块就带大家从vue基础开始认识vue. Vue 是用于构建web界面的库。一般适用于单页应用。 它提供了MVVM数据绑定和一个可组合的组件系统。

单页应用：SPA
    一个页面，页面的构成通过组件来构成。
    单文件组件  a.vue
多页应用：
    n多个页面构成的应用

# 什么是MVC

MVC的定义：MVC是Model-View- Controller的简写。即模型-视图-控制器

# 什么MVVM

  • M model 模型 当前视图中可用的数据
  • V view 视图 页面中的DOM html
  • C controller 控制器 处理加工model
  • viewModel 监控者

# MVVM的由来

MVVM相对于MVC改进的核心思想。在开发过程中，由于需求的变更或添加，项目的复杂度越来越高，代码量越来越大，此时我们会发现MVC维护起来有些吃力，首先被人吐槽的最多的就是MVC的简写变成了Massive-View-Controller（意为沉重的Controller） 实际上MVVM是在MVC 的基础上将MVC 中那个的C-> controller控制器层抽出来的 形成了ViewModel。

# 声明式的渲染

{{}} 可以将Vue实例中的data属性下声明的成员变量{data 对象的属性}的值渲染在指定的位置
1.必须渲染data中声明的成员变量
2.{{}}中除了放置变量 也可以放置固定的值,还可以放一些简单的表达式 (比如三目运算符)，会将表达式的结果渲染
3.标签内可以存在多次声明渲染，不会发生内容的覆盖，会进行内容的拼接

# 指令渲染

v-html：（渲染HTML片段）将指令后的 变量渲染到DOM中 v-text：（渲染文本） 将指令后的 变量渲染到DOM中 两者的区别，同js中的innerHTML 和innerText相同

        <div class="message">{{message}}</div>
        <h1 class="title">{{bol==0?'男':'女'}}</h1>
        <p>{{number*number*number}}</p>
        <h2>{{"asdfasdfasdf"}}</h2>
        <nav>{{username}},{{bol}}</nav>

# 指令渲染与声明渲染的区别：

1.当没有Vue的时候，声明式的渲染会将{{}}表达式以及其中的内容进行渲染；而指令渲染不会对元素进行任何的渲染操作。v-html或v-text 就相当于一个自定义的属性 2.当指令渲染和声明渲染同时存在，那么指令渲染会覆盖当前元素的内容

        <div v-text="title"></div>
        <div v-html="html_"></div>
        <div v-html="bol==0?html_:title">{{message}},初次见面，请多多关照</div>
